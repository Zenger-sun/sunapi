# Contributing

Thank you for considering a contribution to SunAPI Open Source Edition.

## Legal and License Requirements

- Only submit work that you wrote yourself or that you have the right to
  contribute under this repository's licenses.
- Preserve copyright headers, AGPL notices, MIT notices, `NOTICE`, and
  third-party attribution.
- Do not copy proprietary UI, icons, charts, animations, or assets into this
  repository.
- Mark meaningful changes to AGPL-covered frontend files clearly in commit
  messages or pull request descriptions.
- Do not add anti-tamper or anti-user-modification mechanisms that would prevent
  users from exercising open source license rights.

## Security Requirements

- Do not commit credentials, tokens, private keys, `.env` files, local databases,
  or production configuration.
- Use `config.example.yaml` and placeholder values for examples.
- Report vulnerabilities according to `SECURITY.md` instead of opening a public
  issue for sensitive reports.

## Brand Requirements

- Do not use SunAPI names, logos, domains, or release channels to imply an
  unofficial fork is an official release.
- Keep upstream new-api attribution where AGPL-covered frontend code is used.

## Pull Requests

- Keep changes focused.
- Include tests or verification notes for behavior changes.
- Update documentation when changing configuration, public APIs, security
  behavior, or license-sensitive UI surfaces.
